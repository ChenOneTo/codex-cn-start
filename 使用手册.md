# Codex 汉化器 · 使用手册

把官方 **OpenAI Codex 桌面版** 的英文界面实时翻译成中文，并解锁 API Key 模式下被禁用的插件功能。

> 版本 v1.1.0 ｜ 适用：Windows 10/11 + Microsoft Store / MSIX 安装的 OpenAI Codex
>
> 🔑 **内测阶段**：本工具需输入**内测码**激活后方可使用，详见下方第二节。

---

## 一、它能做什么

| 功能 | 说明 |
|---|---|
| **界面汉化** | 菜单、按钮、设置、提示、错误信息等实时翻译成中文（4591 词条 + 369 动态模式） |
| **插件解锁** | API Key 登录模式下，原本被禁用的插件入口 / 插件市场 / 安装按钮全部可用 |
| **完全可逆** | 不修改 Codex 任何文件。关掉本工具，Codex 立刻恢复原样，应用更新也不受影响 |

**重要前提**：必须用本工具启动 Codex 才有汉化和解锁效果。直接点 Codex 桌面图标启动是不带这些功能的。

## 二、快速上手（推荐：用 exe，无需装 Node）

1. 拿到 **`codex-cn-start.exe`**，复制到任意位置。
2. **完全退出 Codex**（如果它正开着）。右键任务栏图标 → 退出，或在任务管理器里结束 Codex 进程。
   > 这一步很关键：Codex 已在运行时，本工具无法接管它。
3. **双击 `codex-cn-start.exe`**。
4. **首次使用需激活内测码**（仅第一次，且需联网）：黑色窗口会提示
   
   ```
   本工具处于内测阶段，需输入内测码激活后使用。
   请输入内测码（形如 CXZH-XXXX-XXXX-XXXX，回车确认）：
   ```
   把作者发给你的内测码粘进去回车。显示 `✓ 内测码激活成功` 即可。
   > 激活信息保存在本机，之后启动**免再次输入**；换电脑需重新激活（每个码可激活的台数有限）。
5. 激活后继续显示：
   ```
   Codex 汉化器 v1.1.0 · 词条 4591 · 模式 369
   插件解锁 · 开(modern)
   启动 Codex（端口 9222）...
   已激活 Codex，PID=xxxxx，等待调试端口...
   ✓ 已注入目标 Codex
   开始注入并持续监听新窗口/页面...
   ```
6. Codex 窗口随之打开，界面已是中文，插件功能可用。
7. **让那个黑色窗口一直开着**。用 Codex 期间它在后台监听（新开的窗口/刷新的页面也会自动翻译）。

**停止汉化**：关闭那个黑色命令行窗口即可。Codex 不受影响，可继续使用（只是变回英文、插件入口恢复禁用）。

## 三、日常使用建议

- **每次想用中文 Codex**：先确保 Codex 没开 → 双击 exe（或 cmd）→ 用它打开的 Codex。
- **想后台运行不占屏**：启动后把黑色窗口最小化即可；或用 `start /min codex-cn-start.exe`。
- **临时只要翻译、不要插件解锁**：用命令行启动并先设环境变量：
  
  ```bat
  set CODEX_ZH_UNLOCK=0
  codex-cn-start.exe
  ```

## 四、常见问题

**Q：内测码相关问题（激活失败 / 换电脑 / 提示已失效）？**
A：

- **激活失败**：核对内测码是否输错；提示「已达激活台数上限」说明该码绑定的电脑已满，「已过期/已被吊销」请联系作者。
- **激活需联网**：仅首次激活与偶尔的后台复检需要联网，平时启动不联网也能用。
- **换电脑/重装系统**：会被视作新机器、占用一个激活名额；名额用尽请联系作者重置。
- **提示「内测授权已失效」**：内测码被吊销或过期，按提示重新输入有效码即可。

**Q：双击图标启动的 Codex 怎么没汉化？**
A：直接点 Codex 图标启动的不带调试端口，本工具接管不了。必须用本工具（exe / cmd）来启动 Codex。

**Q：提示「未能连接调试端口」「请先完全退出 Codex」？**
A：Codex 已经在后台运行了。完全退出 Codex（任务管理器结束所有 Codex 进程）后，再双击本工具。

**Q：杀毒软件 / Windows SmartScreen 报警？**
A：exe 内嵌了 Node 运行时且未做代码签名，属常见误报。
- SmartScreen：点「更多信息」→「仍要运行」。
- 杀毒报毒：把 exe 加入白名单，或改用源码方式（`启动汉化.cmd`）。

**Q：有些文字还是英文？**
A：少量专有名词（Codex、VS Code、GitHub、快捷键、型号名等）按约定保留不译。极少数跨多个界面元素拼接的句子可能只译出片段。也可自行补词典（见下一节）。

**Q：插件解锁没生效 / 插件入口还是灰的？**
A：确认横幅显示了 `插件解锁 · 开`。Codex 大版本更新后解锁特征可能失效，需要更新 `unlock.js`（详见《工作记录.md》维护说明）。

**Q：Codex 更新后还能用吗？**
A：汉化基本不受影响；若新版改了文案，少量界面可能回到英文，补词典即可。插件解锁若失效需更新特征串。

**Q：会不会把我的 Codex 弄坏 / 违反完整性校验？**
A：不会。本工具不改 Codex 任何文件，只在运行时往页面注入脚本，完全可逆。

## 五、自定义词典（进阶，可选）

想补充或修改翻译，**不用重新打包 exe**：

1. 在 `codex-cn-start.exe` 同一目录放一个 `glossary.json`（格式 `{"English":"中文"}`）。
2. 启动时本工具会优先读取 exe 旁边的这个文件，覆盖内置词典。

含占位符的动态句子（如 `Search {query}`）放进同目录的 `patterns.json`，把占位符写成正则捕获组、中文模板用 `$1` 引用。

> 完整待译清单见 `strings_static.json`（静态串）和 `strings_icu.json`（含占位符串）。

## 六、环境变量（高级设置）

| 变量 | 默认 | 作用 |
|---|---|---|
| `CODEX_ZH_UNLOCK` | `1`（开） | 设 `0` 关闭插件解锁，只保留翻译 |
| `CODEX_ZH_LANG` | `zh-CN` | 原生应用菜单（文件/编辑/视图）语言 |
| `CODEX_ZH_PORT` | `9222` | CDP 调试端口（一般无需改） |
| `CODEX_ZH_AUMID` | `OpenAI.Codex_2p2nqsd0c76g0!App` | Codex 应用标识，更新换包名时可能需改 |
